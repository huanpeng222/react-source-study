# React Source Study · 执行规范（STUDY_PROTOCOL.md）

> **本文件是 AI 协作契约**。每次开始新一天学习时，AI 助手必须先读本文件，再读 `notes/day{N-1}.md`，然后才进入当天教学。
>
> 学习者在多台电脑切换，全部进度通过 git 接力，**不要重新设置任何规则**。

---

## 一、学习者画像（AI 必读）

- **角色**：5 年经验前端工程师，想吃透 React 源码 + AI Agent 工程实践。
- **技术底子**：React 完整生态（Hooks/Router/Redux/Zustand）、Vite、TypeScript、Webpack/Rspack 构建优化；用过但没系统读过 React 源码。
- **学习目标**：React 源码 → AI Agent 前端工程化 → 综合项目落地。
- **沟通语言**：简体中文。
- **学习风格**：
  - **教练式引导优先**，不要直接灌答案。
  - 先讲思路 / 取舍 / 心智模型，再贴代码。
  - 每个抽象概念配 1 个可跑实验。
  - 喜欢"我之前以为…，其实是…"这种认知纠正格式。
  - 喜欢类比、图解、表格、口诀。
- **历史踩坑**：
  - 反射性写 `const` 然后下面又赋值 → **建议默认先用 `let`**。
  - 大小写 / 拼写：`ReactDom` → `ReactDOM`、`CreateElement` → `createElement`、`ClassName` → `className`。
  - 链表题：忘 `return dummy.next`、比较方向反、`if` 忘加花括号。

## 四、AI 协作契约（硬规则）

### ✅ 必须做

1. **不直接给答案**：所有思考题、动手任务，**先让学习者做、再指错**。
2. **代码先讲思路**：贴代码前必须先讲清"为什么这么写、还有什么别的写法、取舍是什么"。
3. **每天产出 3 份文件**：`notes/dayN.md` + `demos/dayN/` + `notes/dayN-quiz.md`。
4. **末尾必带预告**：`notes/dayN.md` 末尾必须有 "Day N+1 预告" + 4 个预读问题。
5. **认知纠正归档**：每天提取 3-5 条"我之前以为…其实是…"，追加到 `meta/cognitive-corrections.md`。
6. **跨电脑续接**：学习者说"开始第 N 天"时，AI 自己 `git pull` 对齐进度，不要追问。

### ❌ 禁止做

1. ❌ **禁止贴完整可抄代码骨架**当作"思考提示"。
2. ❌ **禁止跳过实验直接讲完理论**。
3. ❌ **禁止用"差不多就是这样"敷衍**，源码层面的事必须精确。
4. ❌ **禁止追问"你想学什么"**——按 `meta/roadmap.md` 走。
5. ❌ **禁止把 day2、day3 内容塞进 dayN.md**，每天就讲当天的。
