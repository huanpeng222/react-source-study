1 触发更新 · 入队 + 算 lane dispatch → requestUpdateLane → mergeLanes 冒泡

入口：dispatchSetState(fiber, queue, action) 或 dispatchReducerAction(fiber, queue, action) (D6)

Step 1：创建 update = {lane: ???, action, next: null}

Step 2：requestUpdateLane(fiber) 按上下文算 lane (D10)

function requestUpdateLane(fiber) {
  // ① legacy模式 → SyncLane
  if ((mode & ConcurrentMode) === NoMode) return SyncLane;
  // ② render阶段内setState → 借当前wip
  if (renderContext && wipRenderLanes) return pickArbitraryLane(...);
  // ③ startTransition里 → TransitionLane
  if (transition !== null) return requestTransitionLane(transition);
  // ④ 默认 → 事件优先级→lane
  return eventPriorityToLane(resolveUpdatePriority());
}

Step 3：update 入 hook.queue 环链 (D6)

Step 4：markUpdateLaneFromFiberToRoot (D10)

  • fiber.lanes = mergeLanes(fiber.lanes, lane) ← 自己欠了活
  • 沿 return 链到 root：每个 parent.childLanes = mergeLanes(parent.childLanes, lane)
  • 最终：root.pendingLanes |= lane
2 调度 · 选批次 + 启动 workLoop getNextLanes → workLoopSync/Concurrent

批处理 (D6)：同一事件回调里多次 setState → executionContext 标记去重 → 只调度一次

入口：ensureRootIsScheduled(root) → getNextLanes(root) (D10)

nextLanes = getNextLanes(root);
// 从 root.pendingLanes 中取最低位 = 最高优先级批
// getHighestPriorityLane = lanes & -lanes
if (nextLanes & SyncLane) → performSyncWorkOnRoot(root)
else → performConcurrentWorkOnRoot(root)

SyncLane → workLoopSync（不可中断）
TransitionLane → workLoopConcurrent（每 fiber 问 shouldYield()）

3 准备 wip 树 · 双缓存复用 prepareFreshStack → createWorkInProgress

prepareFreshStack(root, renderLanes) (D2)

workInProgress = createWorkInProgress(root.current, null);
// createWorkInProgress 逻辑：
// if current.alternate !== null → 复用（清 flag）
// else → 创建新 fiber + 设 alternate 双向互指

双缓存状态机 (D5)：

  • 初始化：仅 root.current 一个 fiber，alternate=null
  • 首次 render：alternate=null → 新建 wip ← 子 fiber 在 beginWork 逐步创建
  • 首次 commit 后：两棵完整树，全部 alternate 双向配对
  • 后续每次 render：root.current.alternate 已存在 → 直接复用（清 lanes/flags/child 后重用对象）

本质：两块内存反复交换角色，不是每 render malloc 一次。

4 beginWork · 递推（每个 fiber 处理一次） bailout → reconcileChildren → 返回 child 下钻

workLoop 循环每走一步调 performUnitOfWork → beginWork (D4)

bailout 判断 (D4 + D10) — 整棵子树跳过：

oldProps !== newProps → didReceiveUpdate = true  // props引用变→强制重渲
if (!didReceiveUpdate &&
    !includesSomeLane(renderLanes, fiber.lanes) &&      // 自己没活
    !includesSomeLane(renderLanes, fiber.childLanes))    // 子树没活
  → bailoutOnAlreadyFinishedWork(fiber) → 克隆子节点

reconcileChildren (D3) — diff 子节点：

  • 单节点：key+type 匹配 → 复用 fiber；不匹配 → 旧标 Deletion + 新建
  • 多节点：两轮 diff。第一轮从头遍历可复用序列，第二轮 mapRemainingChildren 用 key 匹配 + lastPlacedIndex 贪心算法判断哪些节点需要右移（标记 Placement）
  • 产出 fiber.child = newChild

子 fiber 挂法：长子 → fiber.child；其余子 → sibling 串起来

if child !== null → 返回 child（下次循环处理它，即"下钻"）

Context Provider (D9)：pushProvider 在 beginWork 里做——旧值压栈 + _currentValue = 新值

5 hooks 执行 · 在 renderWithHooks 中 useState/useEffect/useRef/useContext 串联

beginWork 执行函数组件时 → renderWithHooks → 你的组件函数 App(props) 被调用

useState (D6) + lane 过滤 (D10)：

updateReducer(hook) {
  let newState = hook.memoizedState;
  // 遍历 hook.queue.pending 环链
  do {
    update = queue.pending;
    if (isSubsetOfLanes(renderLanes, update.lane)) {
      newState = update.action(newState);  // 应用这个 update
      finishedUpdate = update;
    } else {
      // ★ 优先级不够 → 跳过，挂到 baseQueue 留到下趟
      skippedUpdate = update;
    }
  } while (update !== queue.pending);
  hook.memoizedState = newState;
}

关键：只有 isSubsetOfLanes 返回 true 的 update 才在本趟 render 生效，否则跳过保留下趟。

useEffect (D7)：

// mountEffect / updateEffect
effect = { tag: HookPassive, create, destroy, deps, next: null };
pushEffect(tag, create, destroy, deps);
// deps 没变？areHookInputsEqual=true → 不打 HookHasEffect → 不入调度
// deps 变了？                                    → 入调度
  • effect 对象通过 pushEffect 加入 fiber.updateQueue.lastEffect 环链
  • cleanup（destroy）在下一次 effect 执行前或 fiber 卸载时执行
  • setup（create）在 cleanup 之后执行

useRef (D8)：

// mountRef
hook.memoizedState = { current: initial };
// updateRef — 直接返回 memoizedState（同一对象，跨 render 不变）
return hook.memoizedState;

useMemo / useCallback (D8)：

// mountMemo
hook.memoizedState = [create(), deps];
// updateMemo
if (areHookInputsEqual(nextDeps, prevDeps))
  return hook.memoizedState[0];  // 旧值，bailout
else
  hook.memoizedState = [create(), nextDeps];  // 新值

6 completeWork · 回溯 + 汇总 DOM 属性 diff → bubbleProperties → return 回溯

beginWork 到叶子（child === null）→ completeUnitOfWork (D4)

completeWork 核心逻辑：

  • HostComponent (DOM 标签)：对比 oldProps vs newProps → 标记 Update flag（属性变化）
  • bubbleProperties：把子 fiber 的 lanes/flags 汇总给父
// ReactFiberCompleteWork.js
newChildLanes = mergeLanes(child.lanes, child.childLanes);
parent.childLanes = mergeLanes(parent.childLanes, newChildLanes);
parent.subtreeFlags |= child.flags | child.subtreeFlags;
// flags 汇总是 commit 阶段剪枝的关键：
// commit 只走 subtreeFlags & 对应Flag 的路径

回溯路径：

function completeUnitOfWork(unitOfWork) {
  do {
    completeWork(unitOfWork);
    if (unitOfWork.sibling !== null)
      return unitOfWork.sibling;  // 去兄弟的 beginWork
    unitOfWork = unitOfWork.return;     // 沿父回溯
  } while (unitOfWork !== null);
}

并发中断 (D10)：

  • 时间片到 → shouldYield() = true → workLoopConcurrent 退出循环 → 保留 workInProgress 指针，下次续跑
  • 高优先级插队 → prepareFreshStack 清 workInProgress = null → wip 整棵丢弃

Context Provider 的回溯 (D9)：popProvider 在 completeWork 里弹栈恢复旧值

7 commit · 一把切到屏幕上 beforeMutation → Mutation(改DOM) → Layout → Passive

wip 树完整（root 也 complete 了）→ commitRoot(root)，同步不可中断 (D5)

① beforeMutation：

  • getSnapshotBeforeUpdate / blur focus / scroll
  • 调度 useEffect：scheduleCallback(NormalPriority, flushPassiveEffects)

② Mutation — 真正改 DOM：

  • 遍历 effect 链表，按 fiber.flags 分类处理：
    • Placement → DOM 插入/追加
    • Update → DOM 属性更新
    • Deletion → DOM 移除 + useEffect cleanup 执行
  • root.current = finishedWork — 双缓存切换。这一刻用户看到更新
  • useLayoutEffect 的 cleanup 在此执行

③ Layout：

  • useLayoutEffect 的 setup 同步执行（阻塞浏览器绘制）

④ Passive（commit 结束后异步）：

  • flushPassiveEffects() 执行 useEffect 的 cleanup + setup
  • cleanup 在前，setup 在后，同一宏任务中
8 收尾 · 清账 + 下一轮 markRootFinished → removeLanes → 查看余留

markRootFinished(root, renderLanes) (D10)

root.pendingLanes = removeLanes(root.pendingLanes, renderLanes);
root.suspendedLanes = removeLanes(root.suspendedLanes, renderLanes);
root.pingedLanes = removeLanes(root.pingedLanes, renderLanes);
root.expiredLanes = removeLanes(root.expiredLanes, renderLanes);
// removeLanes = set & ~subset，位差集

检查余留 pendingLanes：

  • 有（被跳过的低优先级 update）→ ensureRootIsScheduled 下一轮
  • 没有 → 等待下一次 setState

同时：root.current 已切换为 finishedWork。旧 current 变成"待复用池"——下次 render 时它的 alternate 指向新 current，createWorkInProgress 会直接复用。
